# Чистый рефакторинг десктопа (clean-rebuild)

> Путь 3: переписываем ПК-версию начисто, блок за блоком, с текущим видом как
> эталоном. Мобилку делаем потом, на чистой основе. Ветка: `clean-rebuild` (от `main`).

## Принципы
- Никаких `!important` и пиксельных сдвигов (`left:-77px` и т.п.).
- Токены — в `tailwind.config.js` (цвета `ink/page/field/line/dark/gold`,
  шрифт `font-tight`, `max-w-container`, `h-header`).
- Каждый блок переписываем на чистый Tailwind, старые классы перестают
  использоваться → потом удаляем мёртвые куски из `index.css`.
- Легаси `index.css` пока подключён; новые компоненты его не используют,
  поэтому конфликтов нет (старые правила цепляются за старые классы/id).
- «Причесать» разрешено: убираем странности, вид ~98% как был.

## Сделано
- **Токены** в `tailwind.config.js`.
- **Шапка** (`src/components/layout/Header.jsx`) переписана начисто:
  - чистый Tailwind, без DOM-хаков (убрано перемещение лого/кнопок в панель),
    без пиксельных сдвигов;
  - `sticky top-0`, контейнер `max-w-container`, высота `h-header`;
  - навигация, поиск, действия (Вход/Регистрация или аватар, «Ищу работу»,
    «Оставить заявку») — сохранена вся логика авторизации;
  - «Услуги» → чистая выпадающая панель (5 категорий + услуги активной),
    вместо мегапанели с фикс-размерами;
  - id `site-header-static` убран → старые правила `#site-header*` в index.css
    стали мёртвыми (на новую шапку не влияют).
  - На узких экранах пока простой бургер (открывает панель услуг) —
    полноценная мобилка позже.
